## 主要更新

### 网址管理功能

全新推出的网址管理功能，让密码管理更加智能便捷。

**核心特性**

- 网址字段：可为每个密码条目添加网站 URL
- 登录后缀：独立配置登录路径（如 `/login`, `/auth`）
- 智能解析：粘贴完整 URL 时自动拆分基础网址和登录后缀
- 一键打开：使用系统默认浏览器直接打开登录页面
- 显示控制：可选择是否在密码卡片上显示网址

**使用示例**

添加密码时粘贴：`https://www.qq.com/login`

系统自动拆分为：

- 网址：`https://www.qq.com`
- 登录后缀：`/login`

查看密码时，卡片显示基础 URL，点击打开按钮后浏览器打开完整登录页面。

---

### 界面优化

- 打开按钮使用外链跳转图标，操作语义更加直观
- 改进密码卡片和列表项的信息展示布局
- 输入网址时自动开启显示选项

---

### 技术改进

**后端增强**

- 新增 `open-url` IPC 通道支持打开外部链接
- 使用 Electron `shell.openExternal()` API 调用系统默认浏览器
- 确保 URL 打开操作的安全性

**数据结构扩展**

```javascript
{
  url: String,        // 网址
  urlSuffix: String,  // 登录后缀
  showUrl: Boolean    // 是否显示
}
```

**兼容性保证**

- 完全向后兼容所有旧版本数据
- 新字段为可选项，不影响现有功能
- 支持降级回旧版本使用

## 使用场景

**快速登录常用网站**

1. 添加密码时填写完整登录 URL
2. 系统自动拆分基础 URL 和登录路径
3. 需要登录时，点击"打开"按钮
4. 浏览器自动打开登录页面
5. 复制用户名和密码完成登录

**管理多个子域名**

```
网站：https://www.example.com
后缀：/admin/login

网站：https://www.example.com
后缀：/user/login
```

不同后缀管理不同的登录入口。

**隐私控制**

敏感网站可以关闭"显示网址"选项，卡片上不显示 URL 信息，但仍可使用一键打开功能。

---

## 注意事项

1. **网址格式**：建议使用完整的 URL 格式（包含 `https://` 或 `http://`）
2. **登录后缀**：后缀会自动拼接到基础 URL 后面，无需重复输入域名
3. **浏览器选择**：使用系统默认浏览器打开，可在系统设置中修改
4. **隐私安全**：网址信息与密码一起加密存储，安全可靠
